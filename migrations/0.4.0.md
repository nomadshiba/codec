# Migration Guide

Breaking changes introduced in `@nomadshiba/codec` **0.4.0**.

---

## Removed: `CodecWithOffsets`

`CodecWithOffsets` (introduced in 0.3.0) has been removed. It was unused and added unnecessary API surface to the base `Codec` class hierarchy.

If you were extending `CodecWithOffsets` to get per-field byte offsets alongside encoded/decoded values, implement the equivalent behavior directly on your
own class instead:

```ts
// Before
import { CodecWithOffsets } from "@nomadshiba/codec";

class TwoFieldCodec extends CodecWithOffsets<TwoFields, TwoFields, TwoOffsets> {
	// ...
	public encodeWithOffsets(value: TwoFields, target?: Uint8Array<ArrayBuffer>) { ... }
	public decodeWithOffsets(data: Uint8Array) { ... }
}

// After
import { Codec } from "@nomadshiba/codec";

class TwoFieldCodec extends Codec<TwoFields, TwoFields> {
	// ...
	public encodeWithOffsets(value: TwoFields, target?: Uint8Array<ArrayBuffer>) { ... }
	public decodeWithOffsets(data: Uint8Array) { ... }
}
```

---

## `Codec.decodeAndReturnValue()` deprecated in favor of `decodeValue()`

`decodeAndReturnValue` is deprecated and will be removed in a future release. Use `decodeValue`, which does the same thing.

```ts
// Before
const value = U32.decodeAndReturnValue(new Uint8Array([0, 0, 0, 7]));

// After
const value = U32.decodeValue(new Uint8Array([0, 0, 0, 7]));
```
