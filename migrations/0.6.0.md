# Migration Guide

Changes introduced in `@nomadshiba/codec` **0.6.0**.

This release contains **no breaking changes** — everything below is backward compatible via deprecated aliases. You can upgrade without touching your code, then
rename at your own pace.

---

## `PaddedEnumCodec` renamed to `FixedEnumCodec`

The `PaddedEnum*` names were renamed to `FixedEnum*` to describe what the codec _is_ (a fixed-size enum) rather than how it achieves it (padding). Behaviour,
wire format, and the underlying implementation are unchanged.

| Old (deprecated)    | New                |
| ------------------- | ------------------ |
| `PaddedEnumCodec`   | `FixedEnumCodec`   |
| `PaddedEnumGeneric` | `FixedEnumGeneric` |
| `PaddedEnumOptions` | `FixedEnumOptions` |

```ts
// Before
import { PaddedEnumCodec, U16, U8 } from "@nomadshiba/codec";
const Event = new PaddedEnumCodec({ Click: U8, Scroll: U16 });

// After
import { FixedEnumCodec, U16, U8 } from "@nomadshiba/codec";
const Event = new FixedEnumCodec({ Click: U8, Scroll: U16 });
```

The old names are still exported as `@deprecated` aliases that point at the new symbols, so existing imports keep working. They will be removed in a future
release — switch to `FixedEnum*` when convenient.

### Note

The two construction-time error messages now use the new name:

- `FixedEnumCodec: variant "<key>" must have a fixed-size codec`
- `FixedEnumCodec: indexer must have a fixed-size codec`

If you match on those message strings, update them accordingly.
