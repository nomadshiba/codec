# Migration Guide

Breaking changes introduced in `@nomadshiba/codec` **0.5.0**.

---

## `encode()` no longer takes a `target` buffer — use `encodeInto()`

`encode(value, target?)` has been split into two methods:

- `encode(value)` — always allocates and returns a new `Uint8Array`.
- `encodeInto(value, target, offset?)` — writes in place into `target` starting at `offset` (defaults to `0`) and returns the number of bytes written.

```ts
// Before
const buf = new Uint8Array(4);
U32.encode(42, buf);

// After
const buf = new Uint8Array(4);
U32.encodeInto(42, buf);
```

---

## `decode()` gains an optional `offset` parameter

`decode(data)` is now `decode(data, offset?)`, defaulting to `0`. Existing calls with a single argument are unaffected.

```ts
// New: decode starting at a byte offset instead of 0
const [value, size] = U32.decode(bytes, 4);
```

---

## Removed: `Codec.decodeValue()`

`decodeValue` has been removed. Destructure the tuple returned by `decode` instead.

```ts
// Before
const value = U32.decodeValue(bytes);

// After
const [value] = U32.decode(bytes);
```

---

## Custom codecs: implement `encoder`/`decoder` instead of `encode`/`decode`

`Codec` now declares `encoder` and `decoder` as the abstract members subclasses must implement. The base class provides concrete `encode`, `encodeInto`, and
`decode` methods that delegate to them. If you have a custom class extending `Codec` directly, you must rename your implementation methods and adopt the
overloaded `encoder` signature (allocating mode when `target`/`offset` are `undefined`, in-place mode otherwise).

```ts
// Before
class PascalStringCodec extends Codec<string> {
	readonly stride: Stride<"variable"> = { kind: "variable" };

	encode(value: string, target?: Uint8Array<ArrayBuffer>): Uint8Array<ArrayBuffer> {
		// ...
	}

	decode(data: Uint8Array): [string, number] {
		// ...
	}
}

// After
class PascalStringCodec extends Codec<string> {
	readonly stride: Stride<"variable"> = { kind: "variable" };

	encoder(value: string, target: undefined, offset: undefined): Uint8Array<ArrayBuffer>;
	encoder(value: string, target: Uint8Array, offset: number): number;
	encoder(value: string, target?: Uint8Array, offset?: number): Uint8Array<ArrayBuffer> | number {
		// ...
	}

	decoder(data: Uint8Array, offset: number): [string, number] {
		// ...
	}
}
```

For fixed-size custom codecs, `FixedCodec<O, I>` remains a marker type (not a base class) — declare `implements FixedCodec<O, I>` alongside `extends Codec<O, I>`
so TypeScript checks that `stride` is `{ kind: "fixed"; size: number }`:

```ts
class DateCodec extends Codec<Date, Date | bigint> implements FixedCodec<Date, Date | bigint> {
	readonly stride: Stride<"fixed"> = { kind: "fixed", size: 8 };
	// ... encoder / decoder
}
```

---

## New: `BigVarIntCodec` / `BigVarInt`

A `bigint`-backed counterpart to `VarInt` with no `Number.MAX_SAFE_INTEGER` ceiling.

```ts
import { BigVarInt } from "@nomadshiba/codec";

const bytes = BigVarInt.encode(300n); // Uint8Array [0xAC, 0x02]
const [value] = BigVarInt.decode(bytes); // 300n
```

---

## New: primitive singletons double as types

Every built-in codec singleton (`U8`, `I32`, `F64`, `Bool`, `Bytes`, `Str`, `VarInt`, …) now also exports a same-named type alias for its decoded output, so
you no longer need `Codec.InferOutput<typeof X>` for the common case:

```ts
import type { U32 } from "@nomadshiba/codec";

// Before
type Id = Codec.InferOutput<typeof U32Codec>;

// After
type Id = U32; // number
```
